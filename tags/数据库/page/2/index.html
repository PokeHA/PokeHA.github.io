《MySQL是怎样运行的》笔记 第六章 B+树索引

  1. 没有索引的查找
    • 在没有索引的情况下，查找记录的操作如下：
    1. 在一个页中查找
      • 假设记录数量较少，全部存储在一个数据页中。
      • 有两种情况：
        1. 以主键为搜索条件：
          • 先对页目录使用二分法找到对应的槽，然后遍历得到目标记录。
        2. 以其他字段为搜索条件：
          • 只能遍历所有记录进行查找。
    2. 在多个页中查找
      • 如果表中的记录存储在多个数据页中，查找分为两个步骤：
        1. 使用双向链表遍历每个数据页，因为页之间不是有序的。
        2. 在每个数据页中对记录进行查找。
      • 问题：
        • 没有索引时，需要遍历所有数据页才能找到目标记录。
  2. 使用索引的情况
    1. 一个简单的索引方案
      • 通过以下步骤优化查找效率：
        1. 让所有数据页按主键有序排列。
        2. 为每个数据页建立目录，目录项存储数据页中最小记录的主键和页号。
        3. 通过二分法在目录中查找目标记录所在页，然后在该页中查找具体记录。
        4. 该目录即为索引。

《MySQL是怎样运行的》笔记 第四章 InnoDB记录结构

  1. 准备工作

    • 不同存储引擎对于数据的存储格式都是不同的
    • 这本书主要讲的是InnoDB的格式

  2. InnoDB页简介

    • InnoDB会把数据划分为多个页
    • 以页作为磁盘和内存之间交互的基本单位
    • 页的大小一般为16KB
    • 一次至少读取16KB内容的数据到内存
      • 同样的，一次最少刷新16KB的数据到磁盘

  3. InnoDB行格式

    • 简介

      • 我们以记录为单位往表中存储数据
      • 每一条记录在磁盘上的存储方式就被称为 行格式 或者 记录格式
      • InnoDB有四种不同的行格式
        1. Compact
        2. Redundant
        3. Dynamic
        4. Compressed

《MySQL是怎样运行的》笔记 第三章 字符集和比较规则

  1. 字符集和比较规则简介

    • 字符集简介
      • 字符集负责处理字符与二进制数据的存储关系
    • 比较规则
      • 字典序也就是比较他们二进制的大小
      • 除了字典序外也有其他的比较规则
    • 重要的字符集
      • ASCII
        1. 128个字符，包括数字、大小写字母、空格等等
        2. 每个字符一个字节
      • GB2132
        1. 包括中文和其他国家的字符
        2. 存在于ASCII中的字符用1个字节存储
        3. 其余的字符用2个字节存储
        4. 也就是变长存储方式
      • GBK
        1. 对于GB2132的字符收集范围做了补充，其余一样
      • Unicode系列
        1. 对地球上所有的字符进行了收录，同样是变长编码
        2. utf-8
          • 1-4个字节存储字符
        3. utf-16
          • 2或4个字节存储字符
        4. utf-32
          • 全用4字节存
