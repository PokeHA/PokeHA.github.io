《MySQL是怎样运行的》笔记 第三章 字符集和比较规则

  1. 字符集和比较规则简介

    • 字符集简介
      • 字符集负责处理字符与二进制数据的存储关系
    • 比较规则
      • 字典序也就是比较他们二进制的大小
      • 除了字典序外也有其他的比较规则
    • 重要的字符集
      • ASCII
        1. 128个字符，包括数字、大小写字母、空格等等
        2. 每个字符一个字节
      • GB2132
        1. 包括中文和其他国家的字符
        2. 存在于ASCII中的字符用1个字节存储
        3. 其余的字符用2个字节存储
        4. 也就是变长存储方式
      • GBK
        1. 对于GB2132的字符收集范围做了补充，其余一样
      • Unicode系列
        1. 对地球上所有的字符进行了收录，同样是变长编码
        2. utf-8
          • 1-4个字节存储字符
        3. utf-16
          • 2或4个字节存储字符
        4. utf-32
          • 全用4字节存

  2. MySQL支持的字符集

    • utf8和utf-8mb4
      • utf8
        1. utf8mb3的别名，依旧用1-3个字节存储字符
      • utf8mb3
        1. 是阉割版，用1-3个字节存储字符
      • utf8mb4
        1. 完整版，用1-4个字节存储字符
        2. 如果要存储更完整的数据的话就用这个，比如emoji表情
    • 字符集的查看
      • 使用SHOW CHARACTER SET命令，可以指定匹配的模式
    • 比较规则的查看
      • 使用SHOW COLLATION命令，可以指定匹配的模式
      • 每种语言都有多个比较规则
      • 还会有个默认的比较规则
        1. 比如utf8的默认比较规则就是utf8_general_ci
        2. ci是case insensitive

  3. 字符集和比较规则的应用

    • 各级别的字符集和比较规则
      • 字符集的级别分为
        1. 服务器级别
        2. 数据库级别
        3. 表级别
        4. 列级别
    • 客户端和服务器通信中的字符集
      • 客户端和服务器通信时需要进行字符集的转换
      • 因为转换来转换去容易出错，所以一般通知设置为一样的字符集
    • 比较规则的应用
      • 如果发现按列排序后不是自己想要的顺序
      • 就可以考虑下更改比较规则
