《图解HTTP》笔记 第一章 了解Web及网络基础

1.1 使用HTTP协议访问Web

当我们在浏览器中输入 URL 并返回显示内容时，实际上使用的就是 HTTP（HyperText Transfer Protocol，超文本传输协议）来进行数据传输。

1.2 HTTP的诞生

  1. 为知识共享而规划 Web
    • 基本理念：通过超文本将互联网连成可互相参阅的万维网（WWW）。
    • WWW 的三项技术：
      1. HTML：文本标记语言。
      2. HTTP：文档传输协议。
      3. URL：指定文档所在地址。
    • WWW 简称为 Web。
  2. Web 的成长时代
    • 各种 Web 客户端和服务器陆续出现，推动了 Web 的发展。
  3. HTTP 的发展历程
    • HTTP/0.9：初始版本，未正式公布为标准。
    • HTTP/1.0：正式标准，仍然在许多服务端广泛使用。
    • HTTP/1.1：目前最主流的版本。
    • HTTP/2.0：新版本，尚未完全普及。

1.3 网络基础 TCP/IP

  1. TCP/IP 协议族
    • TCP/IP 是一系列网络协议的集合，负责管理网络通信。
  2. TCP/IP 的分层管理
    • TCP/IP 协议分为四层：
      1. 网络接口层：负责相邻节点之间的数据传输。
      2. 网际层：实现主机间的逻辑通信，负责选择传输路径（包括 IP 协议）。
      3. 传输层：实现主机进程间的逻辑通信（包括 TCP、UDP）。
      4. 应用层：用户与网络的接口，应用层协议如 HTTP、DNS、FTP 等。
    • 分层设计的好处：
      1. 每层只需要解决其特定的问题，简化了设计。
      2. 分层后，各层之间的接口定义明确，改变某层的内部实现不会影响其他层。
  3. TCP/IP 通信传输流
    • 数据传输过程中包括封装和解包。
  4. HTTP 与 IP、TCP 和 DNS 的关系
    • IP（互联网协议）：负责将数据从一个主机传输到另一个主机。
    • TCP（传输控制协议）：确保数据在传输中完整无误。
    • DNS（域名系统）：将域名解析为 IP 地址。
  5. URL 和 URI
    • URI（统一资源标识符）：用于标识网络资源的标识符。
    • URL（统一资源定位符）：是 URI 的子集，专门用于表示资源的具体地址。
    • URI/URL 格式：
      1. 协议名：使用何种协议访问资源（如 HTTP）。
      2. 登录信息：可选项，指定用户名和密码。
      3. 服务器地址：域名或 IP 地址。
      4. 服务器端口号：默认端口号或指定端口号。
      5. 资源路径：资源所在的路径。
      6. 查询字符串：用于传递参数，以 key=value 的形式。
